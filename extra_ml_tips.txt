Extra Machine Learning Tips for Soil/Crop Data
==============================================

1. Handling Class Imbalance
If some crop types have thousands of rows and others only have a few hundred, your model might become biased towards predicting the common crops. 
- For XGBoost, you can pass weights to the algorithm or rely on 'average='weighted'' in your metrics to evaluate performance fairly.
- Alternatively, you can use techniques like SMOTE (Synthetic Minority Over-sampling Technique) to synthetically balance your training data.

2. Feature Importance
After training your XGBoost model, you can plot "Feature Importance" to see which soil properties or geographical features (e.g., latitude, longitude, altitude) are actually driving the predictions.
- Example code:
  import matplotlib.pyplot as plt
  from xgboost import plot_importance
  plot_importance(model)
  plt.show()

3. Confusion Matrix
Besides the F1 score, a Confusion Matrix is the best way to evaluate multiclass models. It shows exactly which crops the model is confusing with each other. This is incredibly useful for troubleshooting!
- Example code:
  import matplotlib.pyplot as plt
  from sklearn.metrics import ConfusionMatrixDisplay
  
  # Note: le is your LabelEncoder
  ConfusionMatrixDisplay.from_estimator(
      model, x_test, y_test, 
      display_labels=le.classes_, 
      xticks_rotation='vertical'
  )
  plt.show()

4. Hyperparameter Tuning
Instead of guessing parameters like max_depth and learning_rate manually, you can use 'GridSearchCV' or 'RandomizedSearchCV' from scikit-learn. These tools automatically test hundreds of combinations of parameters to find the absolute best settings for your specific data.

5. Prevent Data Leakage
Be careful not to include any columns in your 'X' variables that might act as "cheats" (for example, an ID column that perfectly corresponds to a crop type but won't be available in real life). Only include features the Streamlit user will realistically be able to provide.
