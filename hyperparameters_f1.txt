XGBoost Hyperparameters and F1 Score Guide
==========================================

1. n_estimators (Number of Trees)
- What it does: Tells XGBoost how many decision trees to build in sequence. Each new tree corrects mistakes made by the previous trees.
- How to use it: More trees can make the model more accurate, but too many will take longer to train and might overfit.
- Good starting value: 100 to 300.

2. max_depth (Maximum Tree Depth)
- What it does: Controls how deep (or complex) each individual decision tree is allowed to grow.
- How to use it: A higher depth allows the model to learn highly specific patterns, but it makes the model prone to overfitting. Lower depth forces generalization.
- Good starting value: 3 to 7.

3. learning_rate (eta)
- What it does: Controls how much each new tree impacts the final prediction. A lower learning rate means each tree makes a very tiny correction.
- How to use it: Forces each tree to take tiny baby steps, preventing any single tree from having too much power. It makes the model robust but requires more n_estimators to converge.
- Good starting value: 0.01 to 0.3.

4. objective (The Loss Function)
- What it does: Tells XGBoost what mathematical goal it is trying to achieve.
- Value to use: 'multi:softprob' (outputs probabilities for each crop type, standard for multiclass problems) or 'multi:softmax' (outputs the final class integer).

5. alpha (L1 Regularization)
- What it does: Applies a penalty to the model to keep tree weights small, encouraging the model to ignore useless features.
- How to use it: Increasing this makes the model more conservative.
- Good starting value: 0 (no penalty) or 0.1 to 1 if overfitting.

F1 Score for Multiclass Classification
======================================
The f1_score function assumes binary classification by default. Since you are predicting multiple crop types, you must specify the 'average' parameter:
- average='weighted': Calculates F1 for each crop type, giving more weight to crops that appear most often. (Best for imbalanced data).
- average='macro': Calculates F1 for each crop type and takes the straight mathematical average (treats all crops equally).
- average='micro': Calculates global true positives/false negatives across all classes.
- average=None: Returns an array showing the individual F1 score for every single crop type separately.

Example Fix:
score = f1_score(y_test, y_pred, average='weighted')
